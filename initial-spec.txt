
## 1. System Overview: spdchk

**spdchk** is a high-precision network diagnostic utility written in C. It validates path reachability via ICMP before establishing a TCP stream to measure jitter and latency.

### 1.1 Core Objectives

* **spdchk-server**: A passive listener waiting for diagnostic payloads.
* **spdchk-client**: An active prober that executes a tiered testing sequence (ICMP $\rightarrow$ TCP).

---

## 2. CLI Interface & Parameters

The binary uses `getopt.h` for standard argument parsing.

| Flag | Argument | Description |
| --- | --- | --- |
| `-s` | N/A | Start in **Server** mode. |
| `-c` | `<IP>` | Start in **Client** mode, targeting the specified IP. |
| `-p` | `<Port>` | Target port (Default: 2200). |
| `-i` | `<Count>` | Number of packets to send for the test (Default: 10). |

**Example Usage:**

* **Server:** `sudo ./spdchk -s -p 2200`
* **Client:** `sudo ./spdchk -c 10.0.0.5 -p 2200`

---

## 3. Technical Logic (C Implementation)

### 3.1 Reachability Phase (ICMP)

The client begins by verifying the Layer 3 path.

1. **Socket Creation:** `socket(AF_INET, SOCK_RAW, IPPROTO_ICMP)`.
2. **Packet Construction:** Manually building the `struct icmphdr`.
3. **Checksum:** Implementation of the 1s complement sum for the ICMP header.
4. **Verification:** If no response is received within a 2-second timeout (`SO_RCVTIMEO`), **spdchk** exits with a "Target Unreachable" error.

### 3.2 Measurement Phase (TCP)

Upon successful ICMP reply, the client initiates the TCP test:

1. **Handshake:** Standard `connect()` call to the target port.
2. **Payload:** A series of small timestamps are sent to the server.
3. **Server Action:** The server echoes the data back immediately or acknowledges receipt to allow for RTT calculation.
4. **Timing:** We use `clock_gettime` with `CLOCK_MONOTONIC` to avoid issues with system clock jumps (NTP updates).

---

## 4. Metrics Calculation

The application stores timestamps for each packet $i$ in a struct array.

* **Round-Trip Time (RTT):** 
$$RTT_i = T_{end} - T_{start}$$


* **Jitter (Average Delta):** **spdchk** calculates Jitter as the average of the differences between consecutive RTTs:

$$J = \frac{\sum_{i=1}^{n-1} |RTT_{i+1} - RTT_i|}{n-1}$$


* **Packet Loss:** 
$$\text{Loss} = \left( 1 - \frac{\text{Replies Received}}{\text{Packets Sent}} \right) \times 100$$



---

## 5. Struct & Data Definitions

To ensure cross-platform compatibility between VMs, all network data is packed:

```c
struct spdchk_payload {
    uint32_t seq_num;     // Sequence tracking
    struct timespec ts;   // Departure timestamp
    char padding[32];     // Optional padding for MTU testing
} __attribute__((packed));

```

> **Note:** Because **spdchk** uses `SOCK_RAW` for ICMP, the binary must be run with `sudo` or have `setcap cap_net_raw+ep ./spdchk` applied to it.
